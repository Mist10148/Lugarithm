Procgen everywhere, top-down Automation, dialogue Skip/Next, floating editors, streaming roads, and Finish leg

Phase 0 — Procgen on all levels
- Added LevelLibrary.FromManual() to derive a ProceduralLayoutDefinition from an authored manual route.
- Enabled procedural generation on Tutorial and Level 1 (Molo) using tuned TownGenParams.
- Oton stays unchanged; authored manual/auto remain the deterministic fallback.

Phase 1 — Dialogue Skip / Next that works
- Added visible Next ▶ and Skip ⏭ buttons to the dialogue bar in DialogueOverlayBuilder.
- Refactored DialogueController to factor OnAdvancePressed out of Update() and wire both buttons.
- Implemented SkipConversation: completes reveal flow or fast-forwards the runtime without firing gameplay beats.
- Removed Subtitles gating on the dialogue bar so buttons are always reachable.

Phase 2 — Automation reuses Manual's top-down world
- Introduced IAgentView / IGridSpace / IStopView so ExecutionController drives either iso or top-down presentation.
- GridWorldView and JeepneyAgentView now implement the interfaces; SelfDriveAgent accepts IAgentView.
- Added TopDownGridSpace (builds top-down roads from TownLayout + GridTransform) and TopDownAgentView.
- Added GridTransform shared between GridLayoutProjector rasterization and top-down inversion.
- AutomationDriveController branches on procedural.enabled: builds top-down world + CameraFollow2D for procedural levels, keeps iso fallback for authored mazes.
- Scene builders wired top-down world root, agent, and camera follow in both AutomationDrive and CodeDrive.

Phase 3 — Two distinct floating coding windows + AI chat
- AutomationDriveSceneBuilder now uses BuildBlockWindow / BuildCodeWindow floating panels instead of inline tabs.
- BuildBlockWindow and BuildCodeWindow are draggable via DragWindowHandle on their title bars.
- Code window title bar includes a ✦ AI button that toggles the VibeCodingController copilot window.
- Removed in-scene Block/Code tab buttons and tinting from AutomationDriveController.

Phase 4 — Free-roam streaming + persistent Finish leg
- Added StreamingTownGenerator with deterministic chunk append (baseSeed + chunkIndex) and solvability validation.
- Added RouteVisualBuilder.AppendProcedural() and ManualLayoutProjector.ProjectChunk() for delta projection.
- Added GridModel.AppendChunk() to grow the automation grid without moving the agent.
- ManualDriveController streams new chunks when the jeepney nears the current trunk end.
- Added LegCompletionController and LegCompletionOverlayBuilder; wired into ManualDrive, AutomationDrive, and CodeDrive scenes.
- ManualDrive and AutomationDrive now show the Finish leg button when completable instead of auto-ending.
- Note: Automation grid streaming trigger is scaffolded (AppendChunk exists) but not yet wired into ExecutionController.

Phase 5 — Tests
- Added StreamingTownGeneratorTests (determinism, frontier advance, solvability, terminal-end kind).
- Added GridModelAppendTests (bounds growth, preserved cells, stop registration, out-of-bounds wall).
- Rebuilt all scenes after builder changes.

Test results: EditMode 215/215 passing.
