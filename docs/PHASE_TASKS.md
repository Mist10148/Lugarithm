# Lugarithm — Phase Tasks

> **Status:** DRAFT
> **Version:** 0.3
> **Team:** Cyfer
> **Last Updated:** June 27, 2026

This document breaks development into phases. Phases are sequential at the macro level but individual tasks within a phase can run in parallel. All estimates are rough and subject to revision.

---

## Implemented to Date (June 27, 2026)

The prototype is well past Phase 1. Built and verified (EditMode test suite green):

- **Foundation:** Bootstrap scene + persistent managers (GameManager, SettingsManager, **LocalizationManager**, SceneTransitionManager, BadgeUnlockManager, Almanac), JSON save/load, scene transitions. Scenes are generated by editor scripts (`Lugarithm → Build All Scenes`).
- **Town hubs:** top-down walk-around per level with ambient NPC dialogue (press **E**), board the jeepney to start the drive.
- **Manual Mode:** real-time driving, lane changes, Space brake (**Hold or Toggle**), Coin Drawer fares, breakdown minigames, procedural roadside street dressing.
- **Automation Mode:** block + code editors, step/run/pause/reset/speed, grid pathfinding sim, **one-click Autopilot on every level** (self-drives, tends passengers, collects fares).
- **Mid-drive progression gate:** a mandatory town puzzle interrupts each drive.
- **AI Copilot (Vibe Coding):** in-editor chat with Auto / Ask / Plan / Agent / Refactor modes; post-leg Mentor analytics; Almanac Oracle.
- **Progression:** per-town **badge reveal** on leg completion (placeholder art), currency.
- **Settings:** rebuilt into sections with segmented pill selectors (see Phase 1 below).
- **Localization:** English/Filipino UI via `LocalizationManager` + `LocalizedLabel` (menus, settings, HUD, results, badge). Story/heritage content not yet translated.
